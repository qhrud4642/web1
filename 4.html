br이라는 태그는 줄바꿈을 사용하고 싶을 때 쓰는 태그이다. br태그의 특이함은 닫힘 태그가 따로 없어도 된다는 점이다. 즉 /가 없어도 된다는 뜻이다.

p라는 태그는 문단을 나눌 때 사용하는 태그이다.

p는 br과 다르게 문단의 시작과 끝이 있으므로 닫힘 태그가 있어야한다. 즉 마무리에 /가 있어야 된다는 뜻이다.

p태그의 단점은 정해진 길이만큼 문단이 나눠진 다는 것인데 나중에 css라는 것을 배우면 문단과 문단사이의 길이를 조정 할 수도 있다.

11강 내용

img 라는 태그는 image라는 태그로 말 그대로 이미지를 삽입하는 태그이다. 그러나 태그를 img를 코딩한다해도 어떤 이미지인지 정보가 없으므로 속성이라는 것을 추가해준다. 예를들어 img라는 태그안에 사진파일을 넣으려면 src라는 속성을 넣고 ="coding.jpg"을 하고 그 사진파일 속성의 크기를 조절하려면 width라는 속성안에 크기를 조절한다. <>를 빼고 코드를 짠다면 img src="coding.jpg" width="100%" 이렇게 나온다 img라는 태그안에 속성들(src and width 등)의 순서는 상관이없다.

12강 내용

태그안에 태그가 들어갈 때 그 둘을 부모와 자식 태그라고한다.HTML은 부모와 자식태그가 서로 바뀔 때도 있지만 어느 태그가 부모로 나올 때 항상 같이 자식태그로 나오는 태그들이 몇 개 있다. 그 중 하나는 리스트의 라는 태그인 li이다.li 리스트라는 자식태그로서 부모태그를 가지는데 ul 또는 ol 이라는 부모태그를 가진다. li가 자식태그로오고 ul이라는 부모태그가 올때에는 그냥 순서대로 리스트가 적힌다. 예를들어 ul이라는 부모태그에 li1.김/li li2./li li보 3.li경/li 라는 자식태그가 올 때 1.li1.김/li 태그를 지우면 2.li2.보/li 즉 2.보 라는 1. 이 없는 2부터 나온다. 만약에 ul이라는 부모태그 대신에 ol이라는 태그를 쓰고 li김/li li/li li보 li경/li 라는 태그를 쓴다면 1.김 2.보 3.경 이라는것이 나올 것이고 li김/li 태그를 지운다하여도 2.보가 1.보로 바뀐다. 즉 ul을 부모태그로 하고 중간을 지운다면 순서정렬이 자동으로 되진 않는다. 다만, ul로 태그를한다면 맨 왼쪽에 ●이 나온다. ol로 부모태그를 한다면 중간에 지운다해도 순서정렬이 자동으로 된다. ul은 unordered line 이며 ol은 ordered line의 약자이다.

13강 내용

생활코딩 강의설명 title이라는 태그는 웹사이트를 열 때 웹 페이지의 제목을 나타낸다. 웹 페이지를 열면 웹 제목에 title 태그 안에 쓴 제목이 뜬다. 우리가 코딩을 한글로 작성하고 난 후 저장 후 웹 페이지를 띄울 때 한글이 깨질 때가 있다.그러한 이유는 컴퓨터는 최종적으로 모든정보를 0 or 1로 저장한다. 그 0 or 1을 어떻게 저장할 것 인지에 대한 여러가지 약속들 이있는데 그 약속 중 하나가 "utf-8"이다.그렇다면 우리가 작성한 코딩을 웹 브라우저가 열 때도 "utf-8"로 열어야 문제가 없을 것이다. 그래서 우리가 웹 브라우저에게 "웹 브라우저를 열 때에는 UTF-8로 열어줘" 라고 말하기 위한 코딩은 meta charset="utf-8"이다. 여기서 char는 character라는 약자인데 문자라는 뜻이고 set은 규칙이라는 뜻으로 위의 태그는 "UTF-8로 문서를 읽어라"라고 브라우저에게 얘기를 해준 것 이다. 코딩을 짤 때에는 title과 meta charset="utf-8"은 보통 맨 위에 코딩을 짜는데 그 이유는 그 밑으로 나오는 코딩들과는 차이를 두기 위해서이다. title과 meta charset="utf-8"코드들은 본문을 설명한다. 즉 title은 본문의 제목이 무엇인지 설명하고 meta charset="utf-8"코드는 이 본문 utf-8이라는 방식으로 저장되어 있다고 설명한다. 이 둘이 맨 위에 위치하고 나머지 이제 본문들은 body라는 태그로 묶는다. 이것은 코더들의 약속이므로 이렇게 해야한다. 그리고 본문을 설명하는 태그는 head태그로 묶기로 약속이 되어있다. 즉 본문을 설명하는 title과 meta charset="utf-8"태그들은 head라는 태그로 묶이며 나머지 본문들은 body라는 태그로 묶인다. 이렇게 한다면 html에 있는 모든 태그는 head태그 또는 body태그 하나 아래에 반드시 놓이게 된다.head태그와 body태그는 상위에 있으므로 고위직 태그이다.이 둘보다 최상위에 있는 최고위직 태그는 html이라는 태그이다. 본문을 설명하는 것과 본문을 html이라는 태그가 모두 묶고있다.그리고 이 html태그를 관용적으로 html로서 만들어졌다는 것을 표현하기 위해서 문서의 시작에 아래와 같은 코드를 추가합니다.!doctype html

링크를 달고 싶을 때에는 a라는 태그를 쓰면 된다. a는 anchor 즉 닺이라는 의미로 정보 바다에 정박한다는 시적인 표현이다. 링크를 삽입하기 위해서는 링크 주소가 있어야한다. 따라서 a라는 태그에 href(hyperText Reference)라는 것을 옆에 더 쓴다.(img 태그할 때 src와 마찬가지) a href="주소"를 하면된다. 이렇게 한 후 주소를 클릭하면 웹페이지가 새로 열리진 않고 기존의 웹페이지에서 이동하는데 새로운 페이지에서 이 주소를 열고 싶다면 a href="주소" targer="_blank" 를 쓰면된다. 또 이 주소가 무엇인를 툴팁으로 알려주고 싶다면 a href="주소" targer="_blank" tilte="툴팁내용"을 적으면 된다.

17강 내용

인터넷이 동작하기 위해서는 컴퓨터가 최소 2대가 필요하다. 한 컴퓨터는 웹 브라우저(요규하는 컴퓨터) 다른 한 컴퓨터는 웹 서버(공급하는 컴퓨터)다. 두 대의 컴퓨터는 인터넷으로 연결 되어 있다. 웹 브라우저 주소창에 웹 서버에게 있는 파일을 요구하면 웹 서버는 자신이 가지고있는 것에 대해 브라우저가 요구하는 것을 공급해 준다.

18강 내용

우리가 사실 웹 서버를 운영하는 것은 쉽지 않은 일이다. 항상 컴퓨터가 켜져 있어야한다. 웹 브라우저 즉 요구하는 것이 언제 들어 올지 모르기 때문에 항상 켜져 있어야한다.또 웹 서버라는 프로그램을 배워서 깔아야한다. 또 집 밖에서 웹 서버에 연결 할 수 있게 여러가지 조치를 해야한다. 이러한 것은 쉽지 않다. 따라서 웹 서버를 제공하고 컴퓨터를 켜지 않아도 되며 프로그램을 깔 필요도 없고 집 밖에서 웹 서버에 언제든 연결 할 수 있게 해주는 회사들이 있다. 인터넷이 연결되어 있는 컴퓨터 한 대 한 대를 호스트라고 한다. 웹 서버를 운영하기 위한 컴퓨터 다시 말해 호스트를 빌려주는 회사를 웹 호스팅 업체라 한다. 대표적인 회사가 github(www.github.Welcome)회사이다. 앞서 말했듯이 github 회사는 우리가 홈페이지를 운영할 수 있는 웹 서버를 제공해 준다.

19강 내용

18강의 내용은 웹 서버를 제공하는 회사를 배웠지만, 19강은 웹 서버를 우리가 윈도우에 설치하는 방법을 배울 것 이다. 대표적으로 apache라는 프로그램인데, 사실 apache를 직접 까는 것은 쉽지 않은 일이다. 그래서 쉽게 까는 방법이 있다. 그 방법은 "bitnami wamp stack"(맥은 bitnami mamp stack)이라는 프로그램이다. w는 윈도우 a는 apache m은 MySQL라는 데이터베이스이고 P는 PHP이다. 여기서 우리 일단 apache를 위해서 이 프로그램을 설치한다. 설치 후 "Go to Application"을 클릭하면 웹 페이지가 뜬다. 웹 페이지가 성공적으로 뜬다면 apache를 성곡적으로 설치 한 것이다. bitnami 매니저에서 웹서버를 끈 후 아까 그 웹 페이지를 새로고침 한다면, 웹 홈페이지를 찾을 수 없게 된다. 다시 켠다면 정상적으로 웹 페이지가 뜰 것이다.

20강 내용

19강에서 bitnami 웹 페이지를 켯을 것이다. 웹 페이지 주소를 "http://127.0.0.1/index.html"(맥은 "http://127.0.0.1:8080/index.html". 맥이 8080이 붙는 이유는 맥은 자체로 서버를 가지고있어서 서버를 또 만든다면 컴퓨터가 혼란스러우므로 8080이라는 port 즉 서버를 따로 만들어줘서 원래 있던 서버랑 비교되게 한다.)로 바꾸면 C드라이브에 저장 된 bitnami안에 htdocs의 index.html의 웹 페이지가 나 올 것이다. 18강에서는 github이라는 곳에서 내 서버를 대신 만들어 주었지만 bitnami로 통해 내 서버를 내가 만들었다.127.0.0.1은 내 컴퓨터가 브라우저이면서 서버라는 모든 사람들의 약속된 IP주소이다. 즉, 127.0.0.1/index.html을 친다면 bitnami안에 htdocs의 index.html를 열 것이다. htdocs안에 있는 웹 페이지들이 내가 연 서버이기에 htdocs에 index.html이 있다면 "http://127.0.0.1/index.html"을 주소창에 치면 내가 가동시킨 서버를 웹 페이지로 볼 수 있다. 만약 htdocs에 1.html이 있다면 "http://127.0.0.1/1.html"을 주소창에 치면 1.html을 웹 페이지가 띄울 것이다. 만약 주소창에 쓰지않고 내 컴퓨터에 있는 1.html을 더블 클릭하여 연다면 주소창에 1.html을 띄운 것과 더블 클릭하여 띄운 것은 서로 주소가 다를 것이다. 이유는 "http://127.0.0.1/1.html"은 내 컴퓨터가 1.html이라는 웹 페이지를 요청을(브라우저) 했고 또 내 컴퓨터가 그 페이지를 보여주었지만(웹 서버), 더블클릭 한 것 fill://로 시작되는데 그 의미는 그냥 웹 브라우저가 1.html을 연 것이기 때문에 둘이 서로 다르다. 0http는 Hyper Text Transfer Protocol. Hyper Text는 웹 페이지라는 뜻이고 Transfer은 전송하는 것이고 Protocol은 규약,약속이라는 뜻이다. 즉,웹페이지를 전송하기 위해서 만든 통신 규약이라는 뜻이다. 주소창에 입력된 주소가 http://로 시작하면 웹브라우저는 http 통신규약에 따라서 웹서버에 접속하게 된다. file://로 시작하면 웹브라우저는 파일을 직접 열어서 보여준다.) 따라서 127.0.0.1은 내 컴퓨터가 웹 브라우저이면서 웹 서버(bitnami에 htdocs)가 된다.

21강 내용

20강에서는 컴퓨터 한 대로 웹 서버이면서 웹 브라우저 역할을 했지만, 우리의 목표는 다른 컴퓨터가 내 웹 서버에 접속하여 나의 웹 페이지를 보여주는 것이므로 21강은 이것을 하기위한 강의이다. 20강에서 말했듯이 bitnami안에 htdocs에 서버를 열었다. 그렇다면 다른 사람이 이 웹 페이지를 보기위해서는 무엇이 필요할까? 내 컴퓨터의 IP주소가 필요하다. 내 IP주소는 192.168.123.103이다. 내 핸드폰으로 http://192.168.123.103/index.html(맥은 http://192.168.123.103:8080/index.html)을 친다면 내 컴퓨터 bitnami안에 htdocs의 index.html을 볼 수 있을 것이다.(방화벽 꺼져있어야함)
